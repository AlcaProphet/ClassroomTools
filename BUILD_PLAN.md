# BUILD_PLAN.md

## 项目建设目标

本项目的目标不是一次性交付一个工业级系统，而是通过逐步迭代，构建一个“可运行、可学习、可扩展”的课堂管理助手原型。

最终目标分为三个层次：

1. 完成最小可运行版本（MVP）
2. 补齐核心业务模块
3. 形成可演示、可提交 PR 的学习成果

## 一、当前阶段判断

当前仓库处于设计阶段，核心文档已经明确：

- 产品需求：见 [DesignGuide.md](DesignGuide.md)
- 项目简介：见 [README.md](README.md)
- 贡献与协作方式：见 [CONTRIBUTING.md](CONTRIBUTING.md)

因此，建设路线应首先围绕“可运行闭环”展开，而不是先做完整 UI 与复杂算法。

## 二、建设顺序

### Phase 0：环境准备

目标：保证本地可运行。

完成项：

- 安装 PHP + 本地 Web 运行环境
- 安装 Git 并确认仓库状态
- 建立 SQLite 数据库文件
- 确认目录结构可用

验收标准：

- `php -v` 可执行
- `git status` 可正常识别仓库
- 能启动本地入口页面

### Phase 1：基础骨架

目标：搭出最小可用页面结构。

完成项：

- `index.php` 或首页入口
- `assets/` 静态目录
- `db/` 数据目录
- 页面基础布局
- Bootstrap 引入与基础样式

验收标准：

- 浏览器能够打开首页
- 页面布局具备教师端与学生端入口行为

### Phase 2：教师账号体系

目标：建立最基础的教师身份闭环。

完成项：

- 注册页面
- 登录页面
- 会话保存
- 登出能力
- 首位注册者自动成为管理员

验收标准：

- 注册成功后可写入数据库
- 登录后可进入教师端页面
- 不同账号数据隔离

### Phase 3：班级与学生名单管理

目标：构建课堂的基础数据模型。

完成项：

- 教师创建班级
- 自动生成 6 位班级号
- CSV 导入学生名单
- 学号作为唯一识别键
- 学生名单覆盖与更新逻辑

验收标准：

- 新建班级后可生成房间号
- 导入学生名单可展示
- 同一班级下的数据相互隔离

### Phase 4：发言均衡热力图

目标：实现课堂互动可视化核心模块。

完成项：

- 学生发言计数按钮
- 热力图颜色映射
- 零发言学生识别
- 随机点名逻辑
- 历史记录周归档与只读查看

验收标准：

- 点击学生按钮后发言次数加 1
- 热力图颜色按发言次数正确映射
- 每周一自动归档并清零当前计数

### Phase 5：智能分组器

目标：完成第二大核心功能。

完成项：

- 成绩等级 A/B/C 自动计算
- 阈值配置
- 分组参数设置
- 男女/成绩分层轮流分组算法
- 分组结果展示与导出

验收标准：

- 分组结果不出现明显的性别或排名偏斜
- 每次重分组覆盖并保存最近一次结果
- 学生端能展示当前分组信息

### Phase 6：学生端只读体验

目标：让学生不需要注册即可查看自己的数据。

完成项：

- 入口页输入班级号与学号
- 仅查看本人发言和分组信息
- 班级号与学号输入错误提示

验收标准：

- 学生端能够只读查看本人数据
- 错误输入有明确反馈

### Phase 7：文档与回归检查

目标：保证项目具备可提交状态。

完成项：

- 补完 README 与设计说明
- 检查界面是否一致
- 做最小值回归测试
- 确认 PR 描述和提交记录规范

验收标准：

- 主要功能闭环可验证
- 文档与代码一致
- 可方便他人继续协作

## 三、优先级建议

如果你要按最短路径推进，建议只先完成下面 4 个模块：

1. 教师注册/登录
2. 创建班级并生成房间号
3. CSV 导入学生名单
4. 学生端入口并展示本人信息

这四个模块足以支撑后续发言统计和分组模块的成功开发。

## 四、每个阶段推荐的交付物

- 每个阶段只交付一个“小闭环”
- 每个阶段完成后都要做一次命令验证与页面验证
- 不建议跳过中间验证，避免出现“功能看似都做了，但实际不可用”

## 五、Commit 提交建议

建议使用以下提交前缀：

- `feat:` 新功能
- `fix:` 修复问题
- `docs:` 文档更新
- `refactor:` 代码重构
- `style:` 样式调整

## 六、工程目标

你这次做的目标不是“把全部功能写完”。

你的目标是：

- 先实现一个清晰的项目骨架
- 再把核心闭环跑通
- 最终做到“代码可读、功能可演示、适合 PR 提交”

## 七、后续建议

在实现前，先以“第一个闭环”为目标：

- 教师注册
- 教师登录
- 创建班级
- 生成 6 位班级号
- 导入名单

只要这条链路跑通，你后续再做热力图与分组就会容易很多。
