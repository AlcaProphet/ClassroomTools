# 课堂管理助手 · Vibe Coding 完整设计大纲

> **目标**：生成一个 Web 应用，集成两项或多项独立的教育辅助功能，数据本地持久化，适合教师日常使用。

---

## 1. 整体布局与导航

- 页面顶部为标题栏：`📚 课堂管理助手`，右侧显示当前日期。
- 下方采用 **Tab 切换** 方式，包含两个标签页：
  - `🗣️ 发言均衡`
  - `👥 智能分组`
- 每个标签页独立工作，互不干扰，但共享全局样式和数据存储前缀。

---

## 2. 功能模块一：发言均衡热力图

### 2.1 核心目标
帮助教师可视化追踪每位学生的课堂发言次数，快速识别被提问过少或过多的学生，促进课堂参与公平性。

### 2.2 数据模型
```javascript
{
  students: { "张三": 5, "李四": 0, ... },  // 姓名 → 发言次数
  history: [ { name, timestamp }, ... ]     // 可选：最近发言记录（用于进阶统计）
}
```

### 2.3 界面布局
- **左栏（学生名单）**  
  - 顶部：输入框 + “添加学生”按钮，支持批量粘贴（按换行分割）。  
  - 中部：每个学生以 **大按钮** 展示，按钮上显示姓名和当前次数（如 `张三 3`）。点击按钮，该生计数 +1，并伴有短暂视觉反馈（如闪光）。  
  - 底部：“清空所有数据”按钮（带二次确认）和“新的一周”按钮（将当前数据归档保存，清零计数，保留名单）。

- **右栏（热力图）**  
  - 按发言次数 **从低到高** 排列所有学生。  
  - 每个学生以 **色块** 展示，色块颜色根据发言次数映射：  
    - 最低 20% → 红色（#FF4444）  
    - 中间 60% → 渐变从橙到绿  
    - 最高 20% → 蓝色（#4444FF）  
  - 色块内显示姓名和次数，字体大小随次数增加略微增大。  
  - 色块附加 **闪烁动画**：当某生发言次数为 0 时，边框轻微闪烁。

- **底部统计栏**  
  - 总发言次数、零发言人数、发言最多学生（姓名+次数）。  
  - “随机点名”按钮：优先从零发言学生中随机抽取；若无零发言，则从全体随机抽取。点名结果以弹窗或醒目文字展示。

### 2.4 交互细节
- 数据变更（添加、点击、重置）后，左右两侧及底部统计 **实时刷新**。
- 所有数据自动保存至 `localStorage`，键名 `speaking_data`。
- “新的一周”功能：将当前计数存入 `speaking_history`（按周存储），清零当前计数，学生名单保留。

## 3. 功能模块二：智能分层分组器

### 3.1 核心目标
根据学生的 **性别** 和 **成绩层级** 两个属性，自动将全班分成若干小组，保证每组在性别比例和成绩分布上尽量均衡。

### 3.2 数据模型
```javascript
{
  students: [
    { name: "王五", gender: "男", level: "A" },
    { name: "赵六", gender: "女", level: "B" },
    ...
  ],
  groupSize: 4   // 每组人数
}
```
成绩层级预设为：`A（优）`、`B（中）`、`C（待提升）`。

### 3.3 界面布局
- **上部分（名单管理）**  
  - 表格形式展示已添加学生，每行包括：姓名、性别（下拉选择）、成绩等级（下拉选择）、删除按钮。  
  - 下方提供“添加学生”行（输入姓名 + 两个下拉选择），“批量导入”文本框（支持 CSV 格式：`姓名,性别,等级`）。

- **中部分（分组参数）**  
  - 设置每组人数（数字输入框，默认 4）。  
  - “开始分组”按钮。

- **下部分（分组结果）**  
  - 按组号依次展示每个小组的 **卡片**。  
  - 每张卡片内列出组号、成员姓名，并在姓名旁标注 `♂/♀` 和 `A/B/C` 标签。  
  - 卡片底部显示该组性别统计（男/女各几人）和等级统计（A/B/C各几人）。

### 3.4 核心算法（简要）
1. 按性别分层：将学生分为男、女两个子集。  
2. 在各性别子集内，再按成绩等级分层（A/B/C）。  
3. 从每个层中循环轮流抽取学生填入各组，保证每组人数接近设定值，且各层比例整体均衡。  
4. 若总人数不能被组数整除，余数按顺序分配到前几组。

### 3.5 交互与存储
- 学生数据存入 `localStorage`，键名 `group_students`。  
- 分组结果不自动保存，但提供“导出为文本”按钮（复制每组名单）。  
- 若学生名单为空，点击分组时提示“请先添加学生”。

## 4. 全局技术要求（适用于所有模块）
- 需要有账号体系设计
- 需要有多班级设计 （类似于KAHOOT的房间号）
- 使用例如 PHP 等类似的新手入门级编程语言
- 不过度防御性编程
- 不过度考虑安全性问题
- 不过度考虑并发性问题
- 项目尽可能的简单，以让没有开发经验的初学者能快速上手
- 项目最终运行环境已有一定的安全措施（被结合了 WAF 的 NGINX 代理），部署环境为专门实验用的虚拟服务器

### 4.1 离线与依赖
- 可完全离线运行，指不依赖于其他在线 API。
- 可引用前后端库（类似Bootstrap等），离线引入，不使用任何外部在线的 CDN 资源。    
- 字体使用系统默认无衬线字体。

### 4.2 数据持久化
- 每个模块独立使用 `localStorage` 键名，互不冲突。  
- 数据变更后立即保存，刷新页面自动恢复。  
- 提供“导出所有数据”和“导入所有数据”功能（JSON 文件），便于备份。

### 4.3 视觉与交互风格
- 整体采用 **毛玻璃** 或 **扁平卡片** 风格，柔和配色，适合课堂大屏展示。  
- 按钮具有悬停和点击反馈。  
- 响应式设计，在 1280px 以上宽屏显示最佳，但也能在笔记本屏幕上正常使用。

### 4.4 错误处理
- 所有用户输入做基本校验（如空值、非法字符）。  
- 操作失败时，使用页面内提示（非 alert）显示错误信息，数秒后自动消失。
