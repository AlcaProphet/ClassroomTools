# 课堂管理助手 · 设计思路文档

> **目标**：生成一个 Web 应用，集成两项教育辅助功能。支持教师账号、多班级管理（房间号机制），适合教师日常使用。本项目也是 Vibe Coding 初学者的学习项目。

---

## 1. 整体布局与导航

### 1.1 应用入口与角色

- **教师端**：需登录（用户名 + 密码），登录后可创建/管理班级。
- **学生端**：无需注册或登录，通过"班级号 + 学号"直接访问，功能受限（仅查看）。
- 应用根据登录状态自动区分教师端/学生端界面。

### 1.2 教师端布局

- 顶部标题栏：`📚 课堂管理助手`，右侧显示当前登录教师姓名和"退出登录"按钮。
- 若教师有多个班级，标题栏下方显示班级选择器。
- 进入某个班级后，下方采用 **Tab 切换** 方式：
  - `发言均衡`
  - `历史记录`
  - `智能分组`
- 每个标签页独立工作，互不干扰，数据按班级隔离。

### 1.3 学生端布局

- 顶部显示班级号与当前学生姓名/学号（从输入提交获取，无需登录）。
- 下方显示两个只读视图：本人的发言次数记录、当前所在分组的成员信息。

## 2. 功能模块一：发言均衡热力图

### 2.1 核心目标
帮助教师可视化追踪每位学生的课堂发言次数，快速识别被提问过少或过多的学生，促进课堂参与公平性。

### 2.2 界面布局

**左栏 — 学生名单**
- 每个学生以 **大按钮** 展示，显示姓名、学号和当前发言次数。
- 点击按钮 → 该生计数 +1 → 按钮短暂闪光反馈。
- 每个学生按钮旁提供 **-1 操作**（如小按钮或长按），用于撤销误点。
- 底部提供"清空当前班级数据"按钮，需输入"删除"并确认：**仅清空当前班级的发言计数和历史记录，保留学生名单**（无需重新导入 CSV）。

**右栏 — 热力图**
- 按发言次数 **从低到高** 排列所有学生。
- 每个学生以 **色块** 展示，颜色从红（发言少）渐变至蓝（发言多）：
  - 最低 20% → 红色
  - 中间 60% → 橙 → 黄 → 绿过渡
  - 最高 20% → 蓝色
- 色块内显示姓名和次数，次数越多字体越大。
- 发言次数为 0 的学生色块带闪烁边框，引起教师注意。

**底部统计栏**
- 总发言次数、零发言人数、发言最多学生。
- "随机点名"按钮：优先从零发言者中抽取，结果弹窗展示。

### 2.3 历史记录

**Tab 入口**
- 在教师端新增 `📊 历史记录` Tab，与发言均衡平级。
- 教师可在该页面查看往周的发言数据，回顾课堂参与趋势。

**界面布局**
- 顶部提供**周次选择器**（下拉列表），列出所有已归档的周次，如"第 1 周（9月1日–9月7日）"。
- 系统按自然周自动归档（周一 00:00 至周日 23:59），每周一零点自动将上周数据归档为快照，同时清零当前计数，无需教师手动操作。
- 选中某一周后，下方展示该周的**快照数据**：
  - 热力图（与发言均衡页面样式一致，但为只读）
  - 统计栏：总发言次数、零发言人数、发言最多学生
- 底部可选择"导出本周数据"，下载为文本或 JSON 文件。

**说明**
- 系统在每周一首次有人访问时自动完成归档（若整周无人访问，则在下次访问时补归档）。
- 历史记录仅供教师查看，不可修改。

### 2.4 交互要点
- 任何数据变更后，页面实时刷新，无需手动操作。
- 数据存储于服务端，刷新页面或换设备登录后数据不丢失。
- 随机点名功能依赖本地缓存的学生名单，即使网络中断也可正常使用；名单在页面加载时自动从服务端同步更新。

## 3. 功能模块二：智能分层分组器

### 3.1 核心目标
根据学生的 **性别** 和 **成绩层级** 两个属性，自动将全班分成若干小组，保证每组在性别比例和成绩分布上尽量均衡。

成绩层级由分数自动划分，预设阈值：
- `A（优）`：≥ 85 分
- `B（中）`：≥ 75 分
- `C（待提升）`：< 75 分

教师可在班级设置中手动调整这三个阈值（如将 A 改为 ≥ 90 分）。

### 3.2 界面布局

**上部分 — 名单管理**
- 表格展示已导入学生，每行显示学号、姓名、性别、分数、对应等级。
- 成绩等级由系统根据分数自动计算，表格中显示为标签（如 `A`）。
- 学生数据通过 CSV 文件导入，不提供手动逐个添加。
- 提供"重新导入 CSV"按钮，覆盖当前班级学生列表。

**中部分 — 分组参数**
- 设置每组人数（默认 4 人）。
- 设置成绩等级阈值（默认 A≥85、B≥75，其余为 C），教师可随时调整。
- "开始分组"按钮。

**下部分 — 分组结果**
- 卡片式展示每组，每张卡片包含：
  - 组号、成员列表（姓名旁标注性别和等级标签）
  - 该组性别统计（男/女各几人）
  - 该组等级统计（A/B/C 各几人）
- "导出为文本"按钮，一键复制分组名单到剪贴板。
- **每次"开始分组"会覆盖并保存最近一次结果**，学生端"我的分组"据此显示；可随时重新分组，覆盖上次结果。

### 3.3 核心算法思路
1. 按性别将学生分为男、女两组。
2. 每组内部再按成绩等级（A/B/C）分层。
3. 从各层中轮流抽取学生，依次填入各小组。
4. 这样每组在性别和成绩分布上都大致均衡。

## 4. 全局设计

### 4.1 账号体系

- **仅教师有账号**，学生无需注册。
- 教师可自行注册（用户名 + 密码）；**首位注册者自动成为管理员**。
- 登录方式：用户名 + 密码。
- 预留第三方登录接口（OIDC），未来可接入 微信或者第三方登录 账号。
- 登录状态由服务端 Session 维持。

### 4.2 学生名单管理

- 教师创建班级后，通过 CSV 文件导入学生名单。
- CSV 包含字段：姓名、学号、性别、分数。
- 姓名和学号为必填；性别和分数仅分组模块使用。
- 成绩等级（A/B/C）由系统根据教师设定的阈值自动计算，无需在 CSV 中填写。
- 重新导入 CSV 时，以学号为唯一标识：
  - 已存在的学生：更新姓名、性别、分数，**保留其发言数据**。
  - 新增的学生：自动添加。
  - 未出现在新 CSV 中的学生：从当前名单移除，但**历史发言记录保留**（可在历史记录中查看）。

### 4.3 班级与房间号

- 教师可创建多个班级。
- 每个班级自动生成 **6 位数字班级号**（类似 Kahoot! 的房间码），如 `482951`。
- 教师将班级号告知学生，学生凭"班级号 + 学号"即可访问。
- 每个班级的学生名单独立，互不影响。
- **教师仅可见和管理自己创建的班级**，不同教师之间互不可见。
- 班级管理页面提供"删除班级"入口，需输入班级号确认：**连同学生名单、发言计数、历史记录一并删除**，与发言页的"清空当前班级数据"区分。

### 4.4 技术选型

- **前端**：HTML + CSS + JavaScript（不引入框架，降低学习门槛）
- **UI 库**：Bootstrap（离线引入，不依赖 CDN）
- **后端**：PHP（语法简单，适合新手）
- **数据库**：SQLite（单文件，零配置）
- **运行环境**：NGINX + WAF（已有基础安全防护）

### 4.5 数据存储

- 数据主要存储在服务端数据库中，刷新或换设备不丢失。
- 浏览器 `localStorage` 仅做前端缓存，保存学生名单等基础数据，支持随机点名等有限的离线功能；每次页面加载时自动从服务端同步最新数据覆盖本地缓存。
- 提供"导出所有数据"和"导入所有数据"功能（JSON 文件），用于备份。

### 4.6 视觉风格

- **扁平卡片**风格，柔和配色，适合课堂大屏展示。
- 按钮有悬停和点击反馈。
- 响应式布局：1280px 以上宽屏体验最佳，768px–1280px 笔记本屏幕正常适配，768px 以下手机端需保证核心功能可用（学生端、教师发言按钮、热力图可查看）。
- 手机端优先保证学生端体验，教师端允许部分功能（如分组管理）在宽屏下操作更佳。
- 使用页面内 toast 提示，替代浏览器原生弹窗。

### 4.7 安全原则

- 达到基础安全要求即可，不过度追求生产级标准。
- 密码加密存储，防 SQL 注入，防跨账号数据访问。
- 运行环境已有 WAF + NGINX 提供额外防护。
- 不过度处理：CSRF、Rate Limiting、并发锁等暂不考虑。

### 4.8 设计原则

- 🧒 **极简至上**：代码能跑就行，不追求工业级优雅。
- 🚫 **不防御性编程**：不处理极端边界情况。
- 🔓 **不纠结安全**：运行环境已有基础防护。
- 👥 **不考虑并发**：面向单班级、单教师使用场景。
- 📖 **中文注释**：运行逻辑加中文注释，方便新手理解。

## 5. 学生端设计

学生无需注册或登录，通过浏览器访问带参数的链接：
```
https://example.com/?class=482951
```

> 教师端在班级管理页面提供「复制班级链接」按钮，一键复制上述链接。

### 5.1 访问流程
1. 教师将班级链接分享给学生（也可口头告知班级号）。
2. 学生打开链接后，系统自动预填班级号，学生只需输入学号即可进入。
3. 也可直接访问无参数的地址，在入口页手动输入班级号 + 学号后进入。
4. 学号**仅通过输入框提交，不写入 URL**，避免被遍历泄露。

**错误提示（入口页和链接访问均适用）**
- 班级号不存在 → 提示"班级号未找到，请确认后重新输入"。
- 班级号存在但学号不在该班名册中 → 提示"该班级中未找到此学号，请确认后重新输入"。

### 5.2 学生端界面（只读）

- 顶部显示班级号、学号、姓名。
- **我的发言记录**：当前发言次数（不显示班级排名）。
- **我的分组**：所在组号和同组成员（若尚未分组则显示"未分组"）。

### 5.3 说明

- 学生端仅展示本人数据，无任何操作按钮。
- 班级号是 6 位数字，理论上可被遍历——在当前学习项目中可接受。

## 6. 给 AI 的额外指令

- **代码生成目标**：完整可用，不依赖外部在线服务。
- **注释要求**：关键逻辑添加中文注释，面向零基础新手。
- **PHP 风格**：面向过程，不使用框架，直接操作 SQLite。
- **JS 风格**：ES6+ 原生语法，可以引入类似JQuery等基础模块，不要使用复杂的框架。
- **CSS 风格**：以 Bootstrap 为基础，自定义样式覆盖。
